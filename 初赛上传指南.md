# 短视频文案助手 · 初赛上传指南

## 📋 赛前检查清单

在发帖之前，请确认以下事项：

- [ ] 报名帖已通过审核（初赛作品有效的前提）
- [ ] Demo 文件可以正常打开和体验
- [ ] 准备好至少 3 张开发关键步骤截图
- [ ] 准备好至少 3 个关键任务的 Session ID
- [ ] 准备好产品功能截图（建议 4-6 张）

## 📦 准备上传文件

### 1. Demo 文件
你的 Demo ZIP 包已经准备好了：
- 文件位置：`video-copy-assistant.zip`
- 文件大小：约 20KB 左右
- 使用方法：下载后解压，用浏览器打开 `index.html` 即可

> ⚠️ 社区发帖支持上传 20MB 以内的文件，你的 ZIP 包远小于这个限制，放心上传。

### 2. 截图准备（建议提前准备）

需要准备以下几类截图：

**产品功能截图（展示用，建议 4-6 张）：**
- 首页/主界面截图
- 爆款拆解功能截图
- 风格改写功能截图
- 灵感激发功能截图
- 文案优化功能截图

**开发过程截图（证明用，不少于 3 张）：**
- TRAE 对话界面截图（需求分析阶段）
- TRAE 生成代码的截图
- 功能调试/优化的截图
- 任何能证明用 TRAE 开发的截图

## 🚀 发帖步骤

### 第一步：进入初赛专区
1. 打开 TRAE 官方社区：https://forum.trae.cn/
2. 登录你的账号
3. 进入「TRAE AI 创造力大赛」→「【大赛初赛专区】」
4. 点击「新话题」按钮发帖

### 第二步：填写帖子信息

**标题格式：**
```
【学习工作赛道】短视频文案助手 Demo —— AI 驱动的创意文案工坊
```

**标签选择：**
- 必选：`学习工作`（必须跟报名通过的赛道一致）
- 可选：根据需要添加其他相关标签

### 第三步：填写正文

按照以下结构填写正文（对应模板文件中的内容）：

1. **Demo 简介**
   - 是什么（产品形态）
   - 面向谁（目标用户）
   - 主要功能（2-3 个核心功能，配截图）

2. **Demo 创作思路**
   - 灵感来源
   - 想解决的问题
   - 为什么做这个方向

3. **Demo 体验地址**
   - 上传 ZIP 包后，在正文中说明"附件已上传，下载解压即可体验"

4. **TRAE 实践过程**
   - 开发流程简述
   - 关键步骤截图（不少于 3 张）
   - Session ID（不少于 3 个）

5. **附上报名帖链接**

### 第四步：上传附件
1. 在帖子编辑页面找到上传文件的按钮
2. 上传 `video-copy-assistant.zip`
3. 上传后在正文中适当位置引用或说明

### 第五步：检查并发布
- [ ] 标签是否正确（学习工作）
- [ ] 标题格式是否正确
- [ ] 正文四大模块是否完整
- [ ] 功能截图是否清晰
- [ ] 开发过程截图是否不少于 3 张
- [ ] Session ID 是否不少于 3 个
- [ ] ZIP 包是否已上传
- [ ] 报名帖链接是否已附上

确认无误后点击「发布」！

---

## 💡 提高入围概率的小贴士

1. **截图要清晰**：功能截图最好有标注，让评审一眼看懂
2. **过程要详细**：不要只放结果，把踩过的坑、解决思路也写出来
3. **Session ID 要真实**：这是证明作品由 TRAE 开发的关键
4. **体验要可用**：发布前自己下载 ZIP 包测试一下，确保能正常打开
5. **文案要真诚**：写真实的场景和痛点，不要空泛的套话
